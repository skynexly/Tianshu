# 文件版（单 HTML）打包说明

## 这是什么
把整个天枢城 web 应用打包成**一个可离线运行的单 HTML 文件**。

用途：
- 给 **iOS 用户**（装不了 APK，用 Safari 直接打开本地 html 即可游玩）
- 作为**"作者不再维护后仍可游玩"的存档版**（文件在用户手里，永久可用）

> ⚠️ 注意：文件版解决的是"防跑路"，**不是"防丢数据"**。存档仍存在浏览器 IndexedDB 里，数据管理由用户自己负责。

## 怎么打包
在天枢城根目录执行：

```bash
node build-standalone.js
```

产物：`dist/天枢城-单文件版.html`（约 10~11MB）

## 打包脚本做了什么（build-standalone.js）
1. **内联 CSS/JS**：按 `index.html` 里 `<link>`/`<script src>` 的顺序，把所有 css/js 内联进 HTML。
2. **保留图片转 base64**：世界观图标 `img/worldviews/heartsim.png`、教程头像 `img/tutorial-avatar.jpg`、字体 `fonts/ShareTechMono-Regular.woff2` 内联进去。
3. **剥离大图**：`img/fx/`（特效 13M）、`img/live-bg/`（直播背景 7M）、`img/live-ava/`、`img/covers/`（影视封面 5M）全部剥离，引用替换成 1×1 透明占位，避免裂图。天枢城世界观图标本身就是 base64 内联在 `data/builtin-tianshucheng.js`，不受影响。
4. **fetch 本地文件兜底**：应用里有 `fetch('guide.md')`（×3）和 `fetch('css/style.css')`（×1），`file://` 下会被 CORS 拦。脚本把这两个文件内容预置成内存表 `__FILES__`，并 patch `window.fetch`，命中本地路径直接返回，其余（如 wttr.in 天气）走原生 fetch。
5. **去掉 SW 注册**：`file://` 下 Service Worker 无法注册，移除注册代码。
6. **去掉 manifest**，`apple-touch-icon` 用 icon-192.png 转 base64 保留（iOS 加桌面时的图标）。

## 需要保留内联的图片清单
在脚本 `KEEP_IMAGES` 数组里维护。目前：
- `img/worldviews/heartsim.png`（心动模拟世界观图标）
- `img/tutorial-avatar.jpg`（教程头像）

以后若新增必须内联的图，加进这个数组即可。

## 文件版的已知降级项（预期内，不是 bug）
- 直播间背景 / 主播头像 → 空白（走纯色/首字母兜底）
- 影视封面 → 空白
- 怪谈惊吓特效 → 不显示
- 天气 → 可能拉不到（file:// 下外网请求受限，失败自动降级）
- PDF 导入 → pdf.worker 在 file:// 下可能受限（低频功能）

## 验证要点（每次打包后建议自查）
- `grep -c 'serviceWorker.register'` 应为 0
- `grep -c 'standalone-fetch-shim'` 应为 1
- `img/worldviews/heartsim` 字样应消失（已转 base64）
- 手机浏览器打开，测：进世界观、聊天、存读档（IndexedDB 持久化）、使用指南弹窗
