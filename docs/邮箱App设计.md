# 邮箱 App 设计存档

> 本文档记录邮箱 App 的设计决策、与其它 App 的联动玩法。
> 邮箱定位：用户主动"写、寄、收"的异步信件中心，可承载多种跨 App 玩法，不只服务电台。
> 当前状态：未开发。主屏入口已占位（`appId === 'email'`，点击弹"邮箱开发中"拦截）。

---

## 一、定位与设计原则

- 邮箱是「用户主动产出内容 → 异步寄出 → 延迟收到回应」的载体，区别于聊天（实时）、论坛（公开广播）。
- 它的特色是"延迟感"和"被翻牌"的真实感：寄出去的东西不是立刻有反应，而是过一段时间、在别的场景里被采纳/回信。
- 一封信的基本结构（暂定）：`{ id, to, subject, body, createdAt, status, ... }`，`to` 决定这封信流向哪个玩法。

## 二、玩法：给电台投稿（已确定方向，待 0 号最终拍板细节）

让用户通过邮箱给电台投稿，电台**下期采纳**该投稿，形成"邮箱 → 电台"的跨 App 异步联动。

### 已定的设计点
1. **投给谁**：只能投给「我的」里**已订阅**的电台（订阅 = 生成过详情/收听过）。邮箱选收件电台时，列表来源 = 已订阅电台。
2. **采纳确定性**：一定采纳（下期节目必出现这封投稿），不做"AI 看心情攒几期"。
3. **开放投稿的台**（挂 `submission: true`）：
   - 深夜来信（必开）
   - 娱乐头条
   - 趣味新闻
   - 狗血剧场
   - 犀利锐评
   - 今夜鬼话
   - 音乐人志
   - 这些台共同点：一期是"单元式/拼盘式/读投稿式"结构，投稿能自然作为其中一块内容融入。
   - 不开放的台（路况、点歌、连麦、领域杂谈等）：要么是滚动/无题材形式，要么主题型不适合塞投稿。

### 关键冲突与解法（0 号已定方案）
- **冲突**：投稿"一定采纳"，但开放投稿的台基本都有下期主题。若用户投稿内容与下期主题无关，采纳时会撞车。
- **预告机制实情（已核对代码 `nextGenBlock` 判定，11964 行）**：除新闻类、连线类、全天候滚动类（路况 staticShow）外，**所有台每期都由 AI 产出 `[[next:节目名|简介]]` 动态预告**。所以本玩法开放的 7 个台（深夜来信/娱乐头条/趣味新闻/狗血剧场/犀利锐评/今夜鬼话/音乐人志）里，**除了娱乐头条、趣味新闻是新闻类走 `fixedNext` 固定栏目名外，其余 5 个台每期都有具体的下期预告（节目名 + 简介）**。`fixedNext`/`staticShow` 只是 `_radioNextPreviewOf` 取不到 AI 预告时的兜底。
- **解法（0 号定：玩家自己对齐预告）**：投稿选台时，把该台的下期预告**完整展示给玩家（节目名 + 简介都要给，不只名字）**，由玩家自己判断投稿是否贴合下期主题，而不是系统硬塞或 AI 硬融。
  - 有具体下期预告的台（深夜来信/狗血剧场/犀利锐评/今夜鬼话/音乐人志）：展示「下期节目名 + 简介」，并提示"想投别的主题，可以先去电台把下期预告改掉"。
  - 走固定栏目名的新闻类（娱乐头条/趣味新闻）：展示固定栏目名（如"娱乐快递"），提示"这台是新闻拼盘，不挑具体主题，随便投"。
  - 这样玩家看着预告投稿，采纳时投稿与本期主题天然一致，AI 不需要硬融合。
- 取预告数据复用现有 `_radioNextPreviewOf(prog)`，它返回 `{ name, desc }`（nextShow/nextPreview → fixedNext → staticShow 回退）。投稿界面要把 name 和 desc 一起显示出来。

### 落地思路（技术，待开发邮箱 App 后实现）
- 仿现有跨 App 注入范本 `_radioEchoBlockForForum`（论坛→电台）/ `_radioDetailBlockForPost`（电台→论坛）。
- 数据：邮箱里存"投给某电台的稿件"，带收件电台标识（FM+台名）、内容、采纳状态。
- 电台**续期生成正文时**：新增 `_radioSubmissionBlock(prog)`，扫邮箱里投给"这个台"且未采纳的稿，注入给生成 AI，引导本期把投稿作为一个单元采纳进去。
- 采纳后给那封稿打"已采纳"标记，避免下期重复用。
- **采纳回执（待 0 号确认）**：用户怎么知道投稿被采纳了？候选：① 被动在节目里听到；② 邮箱收到一封"您的投稿已被《xxx》采纳"的回执信。倾向 ② 更有反馈感。

### 待 0 号确认/补充
- 投稿是否一期只采纳一封（多封排队，先投先采）？
- 采纳回执走①还是②？
- 投稿在邮箱里的状态流转（草稿 / 已寄出 / 已采纳）。
