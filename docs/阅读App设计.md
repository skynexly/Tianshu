# 阅读 App 设计存档

> 本文档记录「阅读」App 的概念、架构与生成逻辑，便于跨会话延续开发。
> 当前阶段：概念设计（未动代码）。桌面第三页已放置「阅读」App 占位（点击提示"阅读开发中..."）。

## 〇、整体定位

阅读 App 包含两个部分：

1. **小说**：AI 生成的网文式长篇小说（本文档已细化）。
2. **电子书导入**：用户导入本地 epub/txt 阅读（第二期再啃，难点是章节解析/段落定位，独立于小说功能）。

核心体验定调：**「和 char 一起看书」的氛围感**，不追求产出严谨可追的长篇。
连贯性做到"读着顺"即可，不死磕。

> 共读玩法（邀请 NPC 一起读、划线批注、异步评论）：0 号另有想法，后面再聊，本轮不展开。

## 一、小说：三段式生成（与电台「预览→详情→正文」同源）

| 层级 | 内容 | 复杂度 |
|---|---|---|
| ① 列表 | 刷新推荐 / 关键词搜索，生成一批小说（名 / 作者 / 简介 / 分类等） | 轻 |
| ② 详情 | 生成**核心概念（大纲）** + **目录**（规定每章写什么） | 中 |
| ③ 正文 | 单章 ≈ 2500 字，写完顺手输出**本章概要** | 轻（单章） |

### ① 列表
- **刷新推荐**：无约束，生成一批小说卡片。
- **搜索**：按关键词**现生成**一批匹配题材的小说（不是检索旧库），同一套逻辑多带关键词约束。
- 字段：书名 / 作者 / 简介 / 分类（题材）等。

### ② 详情
- 生成**核心概念（大纲）**：全程不变的骨架。
- 生成**目录**：作用类似大纲，规定每个章节写什么。
- **目录可续接**：100 章只是开头（网文长度），写到尾部可**续生成下一批目录**，无限续写。
  - **用户可手动刷新目录**（重生成后续章节规划）。
  - 数据结构需支持分批目录（记录批次 / 当前写到哪批）。

### ③ 正文（分层记忆，上下文恒定）

生成第 N 章时，喂给 AI：

1. **核心概念 + 目录**（不变骨架）
2. **滚动总结**：每 30 章重压一次的总主线（**永远只有一份**，新总结 = 旧总结 + 这 30 章 重新压缩）
3. **近 10 章概要**：第 N-10 ~ N-1 章的一句话概要（滑动窗口）
4. **上一章正文**：第 N-1 章完整正文（保证语气 / 对话 / 悬念无缝衔接）

写完第 N 章正文后 → **顺手输出本章概要**，存起来供后续使用。

> 取舍：滑动窗口外的章节细节会被压进滚动总结，只剩主线梗概。体验向产品，可接受。
> 超长伏笔（早期埋、很晚收）靠目录/大纲本身写明，不指望概要记住。

## 二、小说：NPC 映射玩法（有趣点）

用户可勾选提供 NPC 资料，把 NPC **映射进小说**——书里的角色能看到该 NPC 的影子。

- **开关**：用户自己勾选要不要开（默认关）。
- **映射强度**：**用户自己选**。
  - 弱（彩蛋/致敬）：特质、习惯、名字变体揉进某角色，似像非像。
  - 中（原型映射）：NPC 作为某角色原型，性格/背景/关系对应，换名换设定。
  - 强（直接入戏）：NPC 本人即主角/重要角色，近似同人/外传。
- **注入时机**：**从②详情就注入**（核心概念+目录生成时就把 NPC 织进去），避免正文阶段硬塞角色导致割裂。

## 三、世界观注入

- 小说默认**可架空**，与玩家世界观解耦。
- 用户可**选择性**喂世界观等资料，不喂也不影响。
